# Задача 1. Что такое гетерогенная параллелизация

Гетерогенная параллелизация — это когда мы ускоряем вычисления, используя разные типы вычислительных устройств вместе: чаще всего CPU + GPU (иногда ещё FPGA/TPU). Идея простая: CPU делает то, что ему “удобно”, GPU — то, что ему “удобно”, и вместе получается быстрее, чем только на CPU или только на GPU.

### 1) Различия CPU и GPU в параллельных вычислениях

#### CPU (процессор):

- Мало ядер (обычно 4–32, иногда больше), но каждое ядро сильное.

- Хорошо справляется с:

  - сложной логикой, ветвлениями (if/else);

  - задачами, где много “разных” шагов;

  - последовательными частями алгоритма.

- Большие кэши, умные предсказатели ветвлений.

#### GPU (видеокарта):

- Очень много “маленьких” вычислительных потоков (тысячи).

- Сильная сторона — одинаковые операции над большим массивом данных.

- Хорошо для:

  - матриц, векторов, фильтров;

  - рендеринга, ML, физики;

  - всего, где можно “раздать работу по элементам”.

Ключевая мысль: CPU = меньше потоков, но умнее; GPU = очень много потоков, но проще логика.

### 2) Преимущества гетерогенной параллелизации

- Скорость: тяжёлые массовые вычисления можно вынести на GPU.

- Эффективность: CPU не “забивается” тем, что GPU делает быстрее.

- Гибкость: разные части задачи можно оптимизировать по-разному.

- Лучшее соотношение цена/производительность на задачах типа “много данных”.

Минус тоже есть: надо учитывать передачу данных CPU↔GPU, иначе можно “убить” выигрыш временем копирования.

### 3) Примеры реальных приложений

- Нейросети / Deep Learning (обучение и инференс на GPU, подготовка данных на CPU).

- Обработка изображений/видео (кодирование, фильтры, распознавание).

- Научные расчёты (моделирование, CFD, молекулярная динамика).

- Финансовые расчёты (Monte Carlo, risk-анализ).

- Игры/3D (рендеринг + физика/частицы).
